WooCommerce Hidden Field Bug Repro Issue #65416

This Playground demonstrates the bug: a field registered with location:'address' and a hidden condition based on another additional_field value never becomes visible, regardless of the select value.

▶ Open in WordPress Playground

Steps to reproduce

  1. Click the button above — WordPress + WooCommerce will boot automatically (takes ~30s)
  2. You will land on the Checkout page
  3. Fill in a name and address in the billing section
  4. Look for the select labeled "Customer type (repro)"
  5. Select "Business"
  6. Observe: the field "VAT Number (should appear only for Business)" never appears
Expected: selecting "Business" should show the VAT Number field via the native hidden JSON Schema condition.
Actual: the field is never rendered, regardless of the select value.

What this repro contains

  • WordPress (latest) + WooCommerce (latest from wordpress.org)
  • A minimal mu-plugin with zero custom CSS or JS — only woocommerce_register_additional_checkout_field()
  • A sample product already in the cart so the checkout is accessible
  • Checkout page configured with the native WooCommerce Checkout Block
